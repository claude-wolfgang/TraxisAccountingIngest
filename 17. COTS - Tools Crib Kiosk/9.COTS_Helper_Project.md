# 9. COTS Helper — QR Label & Inventory System

## Overview

A QR-based identification and inventory management system for ProShop's COTS (Components Off The Shelf) module. Each COTS bin gets a printed QR label that links directly to its ProShop page. A Tampermonkey overlay script provides quick ADD/REMOVE inventory actions when a label is scanned.

## Problem Statement

ProShop's COTS module has no API access and requires manual navigation to find items and adjust inventory. Operators need to search through 197 items across 17 categories to find the right part, then click through multiple screens to record quantity changes with proper PO/WO traceability.

## Solution Architecture

### Layer 1 — QR Labels on Bins
Each COTS item has a P-touch printed label with:
- QR code encoding the direct ProShop URL
- COTS ID (e.g., THI-1)
- Item description

Scanning the QR code with the Nadamoo barcode scanner (which acts as a keyboard input) types the URL into a browser address bar, opening the ProShop COTS detail page instantly.

### Layer 2 — Tampermonkey Inventory Overlay
When a COTS page loads, a Tampermonkey userscript injects a modal overlay with:
- Two large buttons: **REMOVE** (red) and **ADD** (green)
- REMOVE prompts for WO# and quantity
- ADD prompts for Vendor PO# and quantity
- Transactions are queued in localStorage for batch processing

### Layer 3 — Nightly Batch Push (Planned)
A scheduled process that reads the transaction queue and replays entries into ProShop's native inventory checkout form, maintaining full PO/WO traceability in ProShop's system of record.

## ProShop URL Pattern

```
https://traxismfg.adionsystems.com/procnc/ots/[COTS_TYPE]/[COTS_ID]$
```

Examples:
- THI-1 → `.../procnc/ots/THI/THI-1$`
- FAS-9 → `.../procnc/ots/FAS/FAS-9$`
- WOHO-160 → `.../procnc/ots/WOHO/WOHO-160$`

## ProShop Inventory Adjustment Process

ProShop tracks COTS inventory with full transaction traceability:

1. Navigate to COTS item page
2. Click "Inventory Details" or the green checkmark
3. Click **CHECKOUT** button
4. **To add stock:** Enter Vendor PO number, Item#, and Quantity
5. **To remove stock:** Enter WO number and use parentheses `(qty)` or minus `-qty` for quantity
6. Click Save

The Tampermonkey overlay script captures this information upfront so the nightly push can automate steps 1-6.

## Hardware Setup

### Printer
- **Brother PT-P700** connected via USB
- 24mm TZe tape (black on white)
- P-touch Editor 6 with driver installed
- Database merge from CSV: COTS_ID → text, Description → text, URL → QR Code (protocol: QR Code)

### Label Specs
- Tape width: 24mm (0.94")
- Label length: 3.75" (fixed)
- Feed/cut gap: ~1" between labels
- Total tape needed: ~197 × 4.75" = ~936" = ~78 feet = 3 cartridges (26.2 ft each)

### Scanner
- **Nadamoo barcode scanner** — acts as keyboard input, types scanned URL into focused browser address bar

### Print Workflow
1. Open P-touch Editor 6
2. Connect CSV via Database menu
3. Create label template: text objects mapped to COTS_ID and Description, QR code object mapped to URL
4. Set protocol to QR Code (not CODE39)
5. Set tape to 24mm, label length 3.75" fixed
6. Print All Records

## Tampermonkey Script Details

### Installation
1. Install Tampermonkey browser extension
2. Create new userscript
3. Paste contents of `proshop_cots_inventory.user.js`
4. Save — script activates on any `/procnc/ots/*` page

### How It Works
- Script matches URL pattern to detect COTS detail pages
- Extracts COTS ID and type from URL
- Attempts to read item description from page (A.K.A. field)
- Displays modal overlay after 1.5s page load delay
- Transactions stored in localStorage under key `cots_inv_queue`
- Status badge (bottom-left) is clickable to view pending queue

### Transaction Queue Format
```json
{
  "cotsId": "THI-1",
  "cotsType": "THI",
  "action": "remove",
  "reference": "24-0135",
  "quantity": 10,
  "timestamp": "2026-02-13T22:30:00.000Z",
  "url": "https://traxismfg.adionsystems.com/procnc/ots/THI/THI-1$"
}
```

### Keyboard Shortcuts
- **Escape** — Close overlay
- **Enter** — Advance to next field or submit form

## API Limitations

The COTS module is **not exposed** in ProShop's GraphQL API. A full schema scan confirmed available queries are limited to: workOrders, parts, contacts, messages, NCRs, clockPunch, users, workCells. No `cots` or `cotsItem` queries exist. This is why the Tampermonkey UI automation approach is necessary rather than direct API calls.
