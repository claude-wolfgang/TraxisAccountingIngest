# Shop Hub v2.0 — Work Summary (Feb 19, 2026)

## Overview

Shop Hub v2.0 extends the existing FASData Live Dashboard into a unified shop floor communication system. Three major capabilities were added:

1. **CNC program comment reading** via FOCAS (extracts part numbers from running programs)
2. **ProShop ERP integration** (automatic work order lookup by detected part number)
3. **Live messaging system** (machine-specific and shop-wide)

Additionally, the **Service Overseer v1.2** was built to monitor and auto-restart all services.

## Components Built / Modified

### 1. FocasMonitor (C# Windows Service) — Program Comment Enhancement

**Source:** `12. FASData Implementation\FocasMonitor\`
**Deploy:** `C:\FocasMonitor\`

| File | Lines | Changes |
|------|-------|---------|
| `Focas.cs` | 213 | Added `cnc_rdprogdir3` P/Invoke + `PRGDIR3` struct |
| `MonitoringService.cs` | 216 | Added program comment extraction on each poll cycle |
| `Database.cs` | 159 | Added `program_comment TEXT` column to `machine_samples` |

**How it works:**
- After reading the current program number, calls `cnc_rdprogdir3()` to fetch the program directory entry
- Extracts the comment field (up to 48 chars), strips surrounding parentheses
- Stores as `program_comment` in SQLite alongside existing sample data
- Errors are logged but non-fatal — polling continues uninterrupted
- Schema migration: `ALTER TABLE machine_samples ADD COLUMN program_comment TEXT` (safe on existing DBs)

### 2. FASData Dashboard (Python/Flask) — Extended to Shop Hub

**Source:** `FASDataDashboard\`
**Port:** 8070 (`http://10.1.1.71:8070`)

#### `fasdata_live.py` (418 lines) — Main Flask App
- Extended `/api/status` response to include `program_comment`, `detected_part`, and `work_orders` per machine
- Added `/api/messages` endpoints (GET + POST) for machine-specific and shop-wide messaging
- Added `/api/messages/all` endpoint for all recent messages
- Background daemon thread refreshes ProShop WO cache every 5 minutes
- Imports three new modules: `part_parser`, `proshop_client`, `message_store`

#### `part_parser.py` (89 lines) — Part Number Extraction
- Regex-based extraction of 4-5 digit part numbers from CNC program comments
- Pattern: `(?:PN[-:])?(\d{4,5})(-[A-Z]{1,3})?`
- Skips false positives preceded by: OP, TOOL, RPM, FEED, SPEED, SUB, MAIN, T, N, O
- Examples: `"(3847-C FINISH MILL)"` → `"3847-C"`, `"(OP20 ROUGH)"` → `None`
- 11 built-in test cases, all passing

#### `proshop_client.py` (206 lines) — ProShop API Client
- OAuth2 client credentials flow with automatic token refresh
- Reads credentials from `.traxis.env`
- Fetches all WOs for current year via GraphQL (pageSize: 500)
- 5-minute cache TTL, thread-safe with `threading.Lock()`
- **3-tier part number matching:**
  1. Exact match on internal part number
  2. Prefix match (base number without revision, e.g., "3847" matches "3847-C")
  3. Suffix/contains match (strips customer prefixes, normalizes leading zeros)
- Filters to active statuses only: `{active, in process, started}`

#### `message_store.py` (112 lines) — Messaging Backend
- SQLite database: `C:\FASData\shophub.db`
- Schema: `messages` table with id, timestamp, author, machine_id, work_order, message
- Machine-specific messages: `machine_id` set to machine name
- Shop-wide messages: `machine_id` = NULL
- Methods: `add_message()`, `get_messages()`, `get_all_recent()`
- Thread-safe with sqlite3 timeout=5

#### `fasdata_dashboard.html` (1,058 lines) — Kiosk Dashboard UI
- Aztec-themed design (obsidian, terracotta, gold, jade, turquoise palette)
- 1920x1080 kiosk layout with 5 machine cards (T2, M2, M3, M6, M8)
- **Machine card features:**
  - SVG semicircular gauge with animated arc fill
  - Live spindle RPM, feed rate, program number
  - Cutting hours vs available hours
  - Status labels: ON TARGET / BELOW TARGET / CRITICAL / OFFLINE
  - Detected part number + program comment display
  - Active WO list (selectable, persisted in localStorage)
  - Message feed (last 3 messages) + input field
- **Shop-wide message bar** (collapsible, at bottom)
- **Summary bar:** shop average %, active machine count, cutting/available hours
- **Refresh intervals:** 60s data, 15s messages, 1s clock
- Author identification via localStorage prompt

## Data Flow

```
CNC Machines (T2, M3, M6 active)
    | FOCAS protocol (port 8193, 60s polling)
    v
FocasMonitor (Windows Service)
    | writes program_comment + machine data
    v
C:\FASData\monitoring.db (SQLite)
    | read-only
    v
Shop Hub Dashboard (:8070)
    | extracts part number (part_parser)
    | looks up WOs (proshop_client → ProShop GraphQL API)
    | reads/writes messages (message_store → shophub.db)
    v
Browser at http://10.1.1.71:8070
    |
    v (health checks, 60s)
Service Overseer (:8060)
```
