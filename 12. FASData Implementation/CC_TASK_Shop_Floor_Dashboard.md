# Claude Code Task: Shop Floor Display Dashboard

## Overview

Build a full-screen HTML dashboard for a 32" 1080p Samsung TV mounted on the shop floor. This displays CNC machine utilization data from the FASData monitoring system. The display PC has Dropbox synced and the browser will open this file on startup.

## Display Specs

- **Resolution:** 1920x1080 (1080p)
- **Screen size:** 32 inches
- **Viewing distance:** 10-20 feet across the shop floor
- **Environment:** Well-lit machine shop (high brightness, high contrast needed)
- **Input:** None — display only, no mouse/keyboard interaction
- **Refresh:** Auto-refresh every 5 minutes (data updates hourly via Dropbox sync)

## Design Theme: Aztec / Mesoamerican Industrial

Go bold with this. It's a machine shop, not a corporate office. Think:

- **Color palette:** Deep earth tones — obsidian black background, terracotta/burnt orange accents, jade green for "good", turquoise highlights, gold/amber for warnings, blood red for critical
- **Geometric patterns:** Angular Aztec-inspired borders, stepped pyramid shapes, interlocking geometric motifs
- **Typography:** Bold, blocky, industrial — easy to read from 20 feet away. Machine IDs should be HUGE
- **Texture:** Stone or carved feel where appropriate (CSS gradients, subtle patterns)
- **Status indicators:** Gauge-style or bold status blocks with Aztec geometric framing
- **Overall feel:** Ancient temple meets modern factory. Powerful, distinctive, unmistakably NOT generic corporate dashboards

## Data Source

The dashboard reads data from a JSON file generated by `generate_report.py`. The JSON file location:

```
C:\Users\traxi\Dropbox\MACHINE COMM Traxis\FASData\reports\report_data.json
```

BUT — since this is a static HTML file opened locally in a browser, it CANNOT read local files via JavaScript (CORS/security restrictions). 

**Solution options (pick the best one):**

### Option A: Embedded in the HTML report
Modify `send_daily_report.py` to also generate a `dashboard.html` file that has the data embedded directly in a `<script>` tag. The scheduled task regenerates it daily at 7 PM. The browser just needs to reload the file.

### Option B: Tiny local HTTP server
A Python one-liner HTTP server that serves the JSON. The HTML dashboard fetches from localhost. More complex but allows more frequent updates.

### Option C: Inline the data from the SQLite database
A Python script that reads the database, calculates current utilization, and writes a complete self-contained HTML file. Run it via scheduled task every hour to match the DB sync frequency.

**Recommended: Option A or C.** Keep it simple — one HTML file with everything baked in.

## Auto-Refresh

The HTML file must auto-refresh. Since the file is regenerated by a scheduled task:

```html
<meta http-equiv="refresh" content="300">
```

This reloads the file from disk every 5 minutes, picking up any changes from the scheduled task.

## Browser Kiosk Setup

After building the dashboard, also create a batch file `start_display.bat` that:

1. Opens Chrome (or Edge) in kiosk/fullscreen mode pointing to the dashboard HTML
2. Can be placed in the Windows Startup folder for auto-launch on boot

```batch
@echo off
:: Launch dashboard in fullscreen kiosk mode
start "" "C:\Program Files\Google\Chrome\Application\chrome.exe" --kiosk --disable-pinch --overscroll-history-navigation=0 "C:\Users\traxi\Dropbox\MACHINE COMM Traxis\FASData\reports\dashboard.html"
```

If Chrome isn't installed, use Edge:
```batch
start "" "C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --kiosk "C:\Users\traxi\Dropbox\MACHINE COMM Traxis\FASData\reports\dashboard.html"
```
