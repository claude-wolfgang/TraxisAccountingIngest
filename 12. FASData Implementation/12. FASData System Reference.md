**Last Updated:** 2026-02-01

## System Overview

Polls FANUC CNC machines via FOCAS protocol every 60 seconds, stores data in SQLite, syncs to Dropbox hourly.

```
┌─────────────────┐     FOCAS      ┌─────────────────┐
│   CNC Machines  │◄──────────────►│  Collector PC   │
│  (10.1.1.x)     │    Port 8193   │  (WrkStationC)  │
└─────────────────┘                └────────┬────────┘
                                            │
                                   SQLite DB│
                                   C:\FASData\monitoring.db
                                            │
                                   Hourly   │
                                   Sync     ▼
                                   ┌─────────────────┐
                                   │     Dropbox     │
                                   │ MACHINE COMM    │
                                   │ Traxis\FASData  │
                                   └────────┬────────┘
                                            │
                                            ▼
                                   ┌─────────────────┐
                                   │    Main PC      │
                                   │   (Dashboard)   │
                                   └─────────────────┘
```

### Step 5: Install as Service

Run as Administrator:
```cmd
sc create "FocasMonitor" binPath= "C:\FocasMonitor\FocasMonitor.exe" start= auto DisplayName= "FOCAS Machine Monitor"
sc start FocasMonitor
```

Verify:
```cmd

```

### Step 6: Setup Dropbox Sync

Create `C:\FASData\sync-fasdata.bat`:

```batch
@echo off
:: FASData Database Sync Script
:: Copies monitoring database to Dropbox for remote access

set SOURCE=C:\FASData\monitoring.db
set DEST=C:\Users\Superuser\Dropbox\MACHINE COMM Traxis\FASData

:: Create destination folder if it doesn't exist
if not exist "%DEST%" mkdir "%DEST%"

:: Copy database (overwrites existing)
copy /Y "%SOURCE%" "%DEST%\monitoring.db"

:: Log the sync time
echo %date% %time% - Database synced >> "%DEST%\sync.log"
```

**Note:** Update `DEST` path if Dropbox is in a different location.

Create scheduled task (run as Administrator):
```cmd
schtasks /create /tn "FASData Sync" /tr "C:\FASData\sync-fasdata.bat" /sc hourly /st 00:00
```

## Database Schema

SQLite database at `C:\FASData\monitoring.db`:

```sql
CREATE TABLE machine_samples (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL,
    machine_id TEXT NOT NULL,
    machine_name TEXT,
    
    -- Connection status
    connected INTEGER NOT NULL,
    error_message TEXT,
    
    -- Operating mode
    mode TEXT,           -- MEM, MDI, JOG, EDIT, HANDLE, etc.
    run_status TEXT,     -- STOP, STRT, HOLD, MSTR
    motion TEXT,         -- ***, MTN, DWL
    
    -- Program info
    program_number INTEGER,
    main_program INTEGER,
    
    -- Speeds
    spindle_speed INTEGER,
    feed_rate INTEGER,
    
    -- Overrides (percentage)
    spindle_override INTEGER,
    feedrate_override INTEGER,
    
    -- Status flags
    emergency INTEGER,   -- 0=off, 1=on
    alarm INTEGER,       -- 0=none, 1=alarm, 2=battery
    alarm_message TEXT,
    
    -- Axis positions (stored as integers, divide by 1000 for mm)
    axis_x INTEGER,
    axis_y INTEGER,
    axis_z INTEGER
);

CREATE INDEX idx_timestamp ON machine_samples(timestamp);
CREATE INDEX idx_machine ON machine_samples(machine_id);
CREATE INDEX idx_machine_time ON machine_samples(machine_id, timestamp);
```

## Useful Queries

### Today's Utilization by Machine
```sql
SELECT 
    machine_id,
    COUNT(*) as total_samples,
    SUM(CASE WHEN spindle_speed > 0 THEN 1 ELSE 0 END) as running,
    ROUND(100.0 * SUM(CASE WHEN spindle_speed > 0 THEN 1 ELSE 0 END) / COUNT(*), 1) as utilization_pct
FROM machine_samples
WHERE connected = 1
  AND timestamp >= date('now')
GROUP BY machine_id
ORDER BY machine_id;
```

### Daily Utilization Trend
```sql
SELECT 
    machine_id,
    date(timestamp) as day,
    COUNT(*) as samples,
    ROUND(100.0 * SUM(CASE WHEN spindle_speed > 0 THEN 1 ELSE 0 END) / COUNT(*), 1) as utilization_pct
FROM machine_samples
WHERE connected = 1
GROUP BY machine_id, date(timestamp)
ORDER BY day DESC, machine_id;
```

### Current Status (Last Reading per Machine)
```sql
SELECT 
    machine_id,
    timestamp,
    connected,
    mode,
    run_status,
    spindle_speed,
    program_number
FROM machine_samples
WHERE timestamp = (
    SELECT MAX(timestamp) 
    FROM machine_samples ms2 
    WHERE ms2.machine_id = machine_samples.machine_id
)
ORDER BY machine_id;
```

### Shift Hours Utilization (6AM-7PM)
```sql
SELECT 
    machine_id,
    date(timestamp) as day,
    COUNT(*) as samples,
    ROUND(100.0 * SUM(CASE WHEN spindle_speed > 0 THEN 1 ELSE 0 END) / COUNT(*), 1) as utilization_pct
FROM machine_samples
WHERE connected = 1
  AND CAST(strftime('%H', timestamp) AS INTEGER) BETWEEN 6 AND 18
  AND strftime('%w', timestamp) BETWEEN '1' AND '5'  -- Mon-Fri
GROUP BY machine_id, date(timestamp)
ORDER BY day DESC, machine_id;
```

## Troubleshooting

### Service won't start
1. Test manually: `C:\FocasMonitor\FocasMonitor.exe`
2. Check Windows Event Viewer → Application logs
3. Verify all DLLs are present in folder

### Connection failures (error -16)
1. Verify machine is on: `ping 10.1.1.xxx`
2. Verify FOCAS port: `Test-NetConnection -ComputerName 10.1.1.xxx -Port 8193`
3. Check FOCAS enabled on CNC: [SYSTEM] → [EMBED PORT] → [FOCAS2]

### Database not syncing
1. Run `C:\FASData\sync-fasdata.bat` manually
2. Check Dropbox path in script matches actual Dropbox location
3. Verify scheduled task: `schtasks /query /tn "FASData Sync"`

### Machines configured: 0
- machines.json is missing or in wrong location
- JSON syntax error (validate with jsonlint.com)

## Adding a New Machine

1. Get the IP address from Program Transfer Tool or CNC screen
2. Edit `C:\FocasMonitor\machines.json` on collector PC
3. Add entry with `"enabled": true`
4. Restart service: `sc stop FocasMonitor && sc start FocasMonitor`
