# 12. FASData Implementation

## Overview

Custom FOCAS-based machine monitoring solution for Traxis Manufacturing to support productivity-based pay system. Tracks utilization across 9 CNC machines and integrates with ProShop ERP, clock feedback displays, and Airtable automation workflows.

## Decision: Build vs Buy

**Decision:** Build custom solution using FOCAS protocol

**Rationale:**
- Existing Ethernet infrastructure already in place for DNC
- Deep integration needed with ProShop ERP and Airtable
- One-time development cost vs ongoing licensing ($3K-$12K/year)
- More granular data control than commercial options

**Commercial options evaluated:**
- FASData
- Scytec DataXchange  
- MachineMetrics

## How to Find FOCAS Settings on CNC

1. Press **[SYSTEM]** on the control panel
2. Go to **[EMBED PORT]** or similar network menu
3. Look for **[FOCAS2]** section
4. Verify **TCP PORT = 8193** (or note if different)

## Pinning a Static IP on Fanuc 0i-MF (DHCP drift recovery)

If a control's IP has drifted because DHCP CLIENT is enabled, force it static:

1. **SYSTEM → EMBED → COMMON** on the control
2. Cursor to `DHCP CLIENT`, type `0`, press **INPUT**
3. **Power-cycle the control fully.** Fanuc 0i-MF embedded Ethernet has split current-vs-next-boot state — until a power cycle, the *active* DHCP state is still on, and the control will reject edits to the `IP ADDRESS` field with a "DHCP available" message.
4. After reboot, return to **SYSTEM → EMBED → COMMON**
5. Cursor to `IP ADDRESS`, type the desired static IP (e.g. `10.1.1.106`), press **INPUT**
6. Verify `SUBNET MASK = 255.255.255.0` and `ROUTER IP ADDRESS = 10.1.1.1`
7. **Switch to the FOCAS2 tab** (SYSTEM → EMBED → FOCAS2) and verify `TCP PORT NUMBER = 8193`. If it shows `0` or blank, FOCAS is not listening — set it to `8193` and INPUT. The FOCAS2 tab is easy to brush during the multi-tab DHCP/IP edits and a stray INPUT zeroes the port silently. Confirmed on M6 2026-05-06 — ping worked, FOCAS failed because TCP PORT had been zeroed during navigation.
8. **Power-cycle a second time** so the new IP and FOCAS port become active

Test from a shop PC. **Both** of these must pass — ping alone is insufficient because the control answers ICMP without FOCAS:

```powershell
ping <new-ip>                                  # L3 reachability
Test-NetConnection <new-ip> -Port 8193         # FOCAS listening — TcpTestSucceeded must be True
```

**Why this matters for FASData:** every machine in `machines.json` is a static-IP assumption. If any control is left on DHCP, its lease can roll over and FOCAS polling silently breaks. Audit all enabled machines (M2, M3, M6, M8, T2) and either set static or add DHCP reservations for their MAC addresses. Audit also has to check FOCAS2 TCP port — a zeroed port produces the same silent failure as DHCP drift.
