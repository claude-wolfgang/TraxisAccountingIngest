# Breakeven Dashboard Deployment

## File Layout

```
D:\Traxis\dashboards\
  breakeven.html           -- Dashboard HTML (copy from this folder)
  runtime_snapshot.json    -- Generated by aggregator every 15 min
```

The aggregator script and config stay in the project folder:
```
32. Breakeven Dashboard\
  focas_runtime_aggregator.py
  config.json
```

## Prerequisites

- Python 3.12+ with `tzdata` package: `pip install tzdata`
- FOCAS monitor running (`FocasMonitor` service) writing to `C:\FASData\monitoring.db`
- `machines.json` accessible at the path in `config.json`

## Step 1: Create output directory

```cmd
mkdir D:\Traxis\dashboards
```

## Step 2: Copy dashboard HTML

```cmd
copy "D:\Dropbox\MACHINE COMM Traxis\Proshop Automation and Claude Projects\32. Breakeven Dashboard\breakeven.html" "D:\Traxis\dashboards\breakeven.html"
```

## Step 3: Update config.json output path (optional)

Edit `config.json` if needed. The `--out` flag on the command line overrides:
```json
{
  "db_path": "C:\\FASData\\monitoring.db",
  "machines_file": "D:\\Dropbox\\MACHINE COMM Traxis\\Proshop Automation and Claude Projects\\12. FASData Implementation\\focasmonitor\\machines.json",
  "timezone": "America/Chicago"
}
```

## Step 4: Create scheduled task

```cmd
schtasks /create /tn "Traxis - FOCAS Runtime Aggregator" /tr "\"C:\Users\TRAXIS\AppData\Local\Programs\Python\Python314\python.exe\" \"D:\Dropbox\MACHINE COMM Traxis\Proshop Automation and Claude Projects\32. Breakeven Dashboard\focas_runtime_aggregator.py\" --out \"D:\Traxis\dashboards\runtime_snapshot.json\" --history 4" /sc minute /mo 15 /ru TRAXIS /rp * /rl highest
```

Notes:
- `/rp *` will prompt for the user password
- `/rl highest` runs with elevated privileges
- The task runs every 15 minutes whether or not the user is logged in
- To verify: `schtasks /query /tn "Traxis - FOCAS Runtime Aggregator"`
- To run immediately: `schtasks /run /tn "Traxis - FOCAS Runtime Aggregator"`
- To delete: `schtasks /delete /tn "Traxis - FOCAS Runtime Aggregator" /f`

## Step 5: Initial test run

```cmd
cd "D:\Dropbox\MACHINE COMM Traxis\Proshop Automation and Claude Projects\32. Breakeven Dashboard"
python focas_runtime_aggregator.py --out "D:\Traxis\dashboards\runtime_snapshot.json" --history 4
```

Then open `D:\Traxis\dashboards\breakeven.html` in a browser. It should show live data.

## Step 6: LAN sharing (optional)

Share the dashboards folder read-only on the LAN:

```powershell
New-SmbShare -Name "dashboards" -Path "D:\Traxis\dashboards" -ReadAccess "Everyone"
```

Access from any Windows machine on the network:
```
\\TRAXIS-PC\dashboards\breakeven.html
```

Or map as a drive and bookmark in a browser.

## Step 7: Kiosk bookmark

For the shop floor kiosk, add a bookmark or shortcut to:
```
file:///D:/Traxis/dashboards/breakeven.html
```
(local) or the UNC path if accessing over the network.

## Logging

Aggregator logs to `32. Breakeven Dashboard\focas_aggregator.log` (5 MB rotating, 2 backups).

## Monitoring

The dashboard shows "Updated X min ago" in the header. If this exceeds 30 minutes, the timestamp turns amber — indicates the scheduled task may have failed or the DB is unreachable.
